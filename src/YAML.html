YAML¶

YAML 1.2 (3rd Edition)¶

第 1 章 简介¶

数据结构有无数种，但它们都可以用三种基本原语充分表示:

  • 映射(散列/字典) – mappings (hashes/dictionaries)

  • 序列(数组/列表) – sequences (arrays/lists)

  • 标量(字符串/数字) – scalars (strings/numbers)

YAML 中的数据类型：

  • collections (mappings, sequences)

  • scalars

第 2 章 概览¶

2.1 集合（collections）¶

YAML 的集合块使用缩进来表示作用域范围，集合中的每一项在自己单独的一行上开始。 每一个映射（mappings）项使用 “: ” 标记来表示键值对， 每一个序列（sequences）项以 “- ” 开始。 使用 # 符号注释。

Mapping Scalars to Scalars(player statistics)¶
hr:  65    # Home runs
avg: 0.278 # Batting average
rbi: 147   # Runs Batted In
Sequence of Scalars(ball players)¶
- Mark McGwire
- Sammy Sosa
- Ken Griffey
Mapping Scalars to Sequences(ball clubs in each league)¶
american:
  - Boston Red Sox
  - Detroit Tigers
  - New York Yankees
national:
  - New York Mets
  - Chicago Cubs
  - Atlanta Braves
Sequence of Mappings(players’ statistics)¶
-
  name: Mark McGwire
  hr:   65
  avg:  0.278
-
  name: Sammy Sosa
  hr:   63
  avg:  0.288

YAML 也支持流式写法（flow styles），使用显式指示符而不是缩进来表示范围。 流式序列语法为为方括号内逗号分隔的列表。 以类似的方式，流式映射使用花括号。

Sequence of Sequences¶
- [name        , hr, avg  ]
- [Mark McGwire, 65, 0.278]
- [Sammy Sosa  , 63, 0.288]
Mapping of Mappings¶
Mark McGwire: {hr: 65, avg: 0.278}
Sammy Sosa: {
    hr: 63,
    avg: 0.288
  }

2.2 结构体（Structures）¶

重复的节点（对象）首先由锚点（用“&”标记）标识， 然后被命名为别名（使用“*”引用）。

Single Document with Two Comments¶
---
hr: # 1998 hr ranking
  - Mark McGwire
  - Sammy Sosa
rbi:
  # 1998 rbi ranking
  - Sammy Sosa
  - Ken Griffey
Node for “Sammy Sosa” appears twice in this document¶
---
hr:
  - Mark McGwire
  # Following node labeled SS
  - &SS Sammy Sosa
rbi:
  - *SS # Subsequent occurrence
  - Ken Griffey
